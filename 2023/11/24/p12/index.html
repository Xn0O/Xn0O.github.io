1.1.1.JDK&JRE&JVM

首先介绍一下它们仨：
由图就可以知道它们的关系是JDK包含JRE包含JVM

  • JDK（开发工具包）：Java标准开发包，它提供了编译、运行Java程序所需的各种工具和资源，包括Java编译器、Java运行时环境、以及常用的Java类库等。
  • JRE（运行环境，运行时类库）：Java运行环境，用于解释执行Java的字节码文件。
  • JVM（Java虚拟机）：Java虚拟机，是JRE的一部分。负责解释执行字节码文件，是可运行java字节码文件的虚拟计算机。

联系和区别

  • JDK包含JRE,而JDK和JRE中都包含了JVM。JDK除了包含JRE还包含一些常用开发工具和基础类库。
  • JDK用于开发，JRE用于运行java程序。
  • JVM是java编程语言的核心并且具有平台独立性。可跨平台。


java程序通过JVM可以在不同的平台运行。

在Java中，Thread thread = new Thread(thread2); 这行代码是创建一个新的线程，其中thread2是这个新线程将要运行的Runnable对象。

更具体地说，这行代码是在创建一个新的线程thread，并将已经存在的thread2作为参数传递给这个新线程。这个新线程将会执行thread2的run方法。

需要注意的是，创建线程时，Java需要一个实现了Runnable接口的对象来指定线程需要运行的代码。在这个例子中，我们已经有一个实现了Runnable接口的thread2对象，所以我们可以将其传递给新的线程。

此外，还需要注意的是，这行代码只是创建了一个新的线程，但并没有启动这个线程。要启动这个线程，你需要调用thread3.start();方法。调用start方法后，新创建的线程就会开始运行。

其他方法

  • sleep()：使当线程进入阻塞状态，在指定时间内不会执行，自动苏醒
  • wait()：不会主动苏醒，需要另外的线程调用notify()/notifyAll()方法唤醒
  • t.join()：只会使主线程（或者说调用t.join()的线程）进入等待池并等t线程执行完毕后才会苏醒，这里的t是定义的线程的名字，如可以thread1.join()
  • interrupt()：中断本线程
  • notify()/notifyAll()：唤醒在某些条件下等待的线程

sleep() && wait() 区别

  • 相同点

    一旦执行后，就会使当前线程进入阻塞状态

  • 不同点

    1.方法声明方式不同，sleep在Thread类中声明，wait在Object类中声明
    2.调用条件不同，sleep可以在任何需要使用的场景下调用，wait必须使用在同步代码块或同步方法中
    3.是否释放锁，sleep不会释放，wait会释放

  2. public boolean isFile()判断是文件还是目录
  3. public boolean isDirectory()判断是文件还是目录
  4. public String getName()返回文件名或目录名
  5. public String getPath()返回文件或目录的路径。
  6. public long length()获取文件的长度
  7. public String[] list()将目录中所有文件名保存在字符串数组中返回。
    File类中还定义了一些对文件或目录进行管理、操作的方法，常用的方法有:
